AIに競合14チャンネルを丸裸にさせた話 — 量産型は負けていた


切り抜きチャンネルを始めた。競合が13個いた。

ゲーム実況の切り抜きチャンネルを始めました。

人気YouTuberの長尺動画から面白いシーンを切り出して、ショート動画やハイライトにする。いわゆる「公認切り抜き」というやつです。

チャンネルを開設して、まず同ジャンルの公認チャンネルを調べました。

**13チャンネルありました。**

最大手は登録者5.7万人、動画1,458本。累計再生9,000万回超。こちらは登録者0人、動画0本からのスタートです。

「これ、勝てるのか？」

感覚で悩んでも仕方ないので、全チャンネルのデータをAIに取得・分析させることにしました。


まず全部のデータを引っこ抜いた

YouTube Data APIを使えば、チャンネルIDさえわかれば登録者数・総再生数・動画本数が取れます。

13チャンネル分のIDを集めて、Pythonスクリプトを1本書きました。30行くらいのシンプルなもので、APIにIDを投げてJSONで受け取るだけです。

```python
# 各チャンネルの統計を取得
for ch_id in channel_ids:
    res = youtube.channels().list(
        part="statistics",
        id=ch_id
    ).execute()
    stats = res["items"][0]["statistics"]
    # subscriberCount, viewCount, videoCount が取れる
```

実行したら数分で14チャンネル分（競合13＋自分）のデータが揃いました。

ここまでは普通です。面白くなるのはこの先。


「1本あたり再生数」を出したら景色が変わった

登録者数で並べると、当たり前に大手が上に来ます。それだけでは何もわかりません。

ふと思いついたのが、**「1本あたりの再生数」を計算してみること**でした。

総再生数÷動画本数。シンプルな割り算です。

```python
views_per_video = total_views / video_count
```

この数字で並べ直した瞬間、それまで見えていなかったものが見えました。

- **A（アルバム型）**：653本 / 登録者2.1万人 → 1本あたり **13万再生**
- **B（シアター型）**：203本 / 登録者1.8万人 → 1本あたり **8万再生**
- **C（老舗最大手）**：1,458本 / 登録者5.7万人 → 1本あたり **6.2万再生**
- D（小話型）：559本 → 5.7万再生
- E（ROOM型）：870本 → 4.3万再生
- F（ニッチ特化型）：355本 → 4.0万再生
- G（ショート専門）：1,088本 → 3.8万再生
- H（放送型）：461本 → 3.5万再生
- I（ポケット型）：643本 → 3.3万再生
- J、K（ファン向け等）：各2.1万再生
- **L（量産型）**：1,681本 / 登録者1.35万人 → 1本あたり **1.3万再生**
- **M（超量産型）**：2,303本 / 登録者1.55万人 → 1本あたり **8千再生**
- 自分：31本 / 登録者23人 → 503再生

最初に目が行ったのはAとMの差です。

Aは653本で1本あたり13万再生。Mは2,303本で1本あたり8千再生。

**Mの方が3.5倍多く投稿しているのに、1本あたりの効率は16分の1。**

え、量産しても伸びないの？


登録者数ランキングは嘘をつく

もうひとつ驚いたのが、Bです。

Bは203本しか投稿していません。14チャンネル中、2番目に少ない。でも1本あたり8万再生で全体2位。

逆にLは1,681本（8倍以上の投稿数）で1.3万再生/本。Bの6分の1の効率です。

登録者数だけで見るとBは5位（1.8万人）、Lは12位（1.35万人）で、どちらもパッとしないように見えます。でも「1本あたり再生数」で見ると、**Bは明らかに勝っている**。

同じ切り抜きチャンネルなのに、なぜこんなに差が出るのか。

データを見た瞬間の疑問をそのままAIに投げました。


AIに「上位と下位の違い」を分析させた

14チャンネルのデータに加えて、各チャンネルの概要欄・動画タイトルの傾向・投稿頻度をAIに渡して、「上位チャンネルと下位チャンネルで何が違うのか」を分析させました。

AIの回答をざっくりまとめると、こういう構造が見えてきました。

**上位チャンネルに共通するもの**

Aは「名場面アルバム」形式です。1本の動画に複数のシーンをまとめて、テーマを持たせて編集している。元動画をただ切り出しただけではない。「構成」という付加価値がある。

Bは「感動・ドラマ系」に完全特化しています。扱うシーンの種類を絞り込むことで、「このチャンネルを見れば泣けるシーンに出会える」という期待値が生まれている。本数は少ないけど、見つけた人は登録する。

Cは単純に蓄積の力です。2021年から5年間、コツコツ投稿してきた老舗。累計9,000万再生は「5年×1,458本」の積み上げ。新規が同じ戦略で追いつくのは非現実的。

**下位チャンネルに共通するもの**

LとMは「量で勝負しようとしている」のが共通点でした。

Lは月32本投稿。ほぼ毎日です。全メンバー・全シリーズをカバーする総合型で、**広く浅く速く**が方針。でも1本あたり1.3万再生。

Mに至っては2,303本で全チャンネル中最多投稿。**本数だけなら圧倒的1位。でも1本あたりの効率は最低。**

AIが言ったのはシンプルな結論でした。

「上位は**編集に付加価値がある**。下位は**素材をそのまま切っているだけ**。」


サムネ317枚をAIに食わせた

データ分析だけじゃ足りないと思って、8チャンネル分のサムネイル画像も集めました。各チャンネル40枚、合計317枚。

これをEmbedding（AIで画像を数値ベクトルに変換する技術）で分析しました。

やったことはシンプルです。317枚を全部ベクトル化して、チャンネルごとに「サムネの重心」を計算する。重心同士の距離を測れば、「どのチャンネルとどのチャンネルのサムネが似ているか」が数値でわかります。

自分のサムネも同じようにベクトル化して、競合のサムネ群と比較しました。

**見えたこと：**

上位チャンネルのサムネには明確な共通パターンがありました。**キャラクターが画面の80〜90%を占めている**。煽りテキストが大きい。

自分のサムネを見返したら、キャラが画面の約68%でした。数字にすると12〜22ポイントの差。でもサムネ一覧で競合と並べると、**自分のやつだけキャラが小さくて埋もれる**。

これは数字を出す前は気づいていませんでした。「なんかパッとしないな」とは思っていたけど、理由は「キャラが小さい」だった。


じゃあ自分はどう戦うのか

データが揃ったところで、戦略を考えました。

**やってはいけないこと：**

- Cの真似（老舗の蓄積には追いつけない）
- LやMの真似（量産しても効率が下がるだけ）
- Gの真似（ショートだけでは中位止まり）

**やるべきこと：**

Aのような「編集付加価値」とBのような「テーマ特化」。でもAは653本、Bは203本。手作業でやると膨大な時間がかかります。

ここで考えたのが、**AIパイプラインの速度を武器にすること**でした。

手動で切り抜きを作っている競合チャンネルは、元動画が公開されてから切り抜きを投稿するまで、早くて翌日、通常は数日かかります。動画を見て、面白いシーンを選んで、編集して、サムネを作って、メタデータを書いて、アップロードする。全部人間がやっている。

一方、AIパイプラインなら元動画の公開から数時間で投稿できます。字幕生成、シーン候補抽出、サムネ自動生成、メタデータ設定まで自動化済み。人間がやるのはシーンの最終選定と品質チェックだけ。

つまり、量産型の「速さだけ」でも、シアター型の「質だけ」でもない。

**速さと質を両立できるのが、AIを使うアドバンテージ。**

具体的には、ショート動画で認知を取って、ハイライト動画（8分超）で広告収益を稼ぐ。ショートは拡散力が強いけどお金にならない。8分超の動画は中間広告が入るので収益になる。この2段構えでいく。


14チャンネル分の分析が1日で終わった

まとめると、やったことは以下です。

- YouTube Data APIで14チャンネルの統計を一括取得（数分）
- 「1本あたり再生数」で全チャンネルを並べ替え（数秒）
- AIに上位・下位の特徴を分析させた（数分）
- サムネ317枚を収集してEmbeddingで傾向分析（数十分）
- データに基づいて戦略を策定

全部合わせて**1日**で終わりました。

もし手動でやっていたら、13チャンネルの動画を片っ端から見て、サムネを目で比較して、Excelに数字を打ち込んで——何日かかるかわかりません。

そしてたぶん、「1本あたり再生数」という指標を思いつかなかったと思います。登録者数で並べて「やっぱり大手には勝てないな」で終わっていた。

**データを出すと、問いが変わる。** 「勝てるか」ではなく「どこで勝てるか」になる。

チャンネルはまだ31本、登録者23人。始まったばかりです。

でも、どこを狙うかは決まりました。


次回は「1万再生なのに登録者23人だった話」を書きます。データで競合を知った次は、データで自分を知る話です。
