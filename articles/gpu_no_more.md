# RTX 4060 Ti買った翌週に、GPUなしで全部できると気づいた話

## きっかけは「Claude Codeの費用を稼がなければ」だった

Claude Codeを使い始めて、気づいたら月に数万円の費用がかかっていました。

便利だから使う。使うから費用がかかる。費用がかかるから何とかしなければいけない。

「このツールに払っているお金を、このツールで稼げないか」という逆転の発想で、ゲーム実況動画の切り抜きチャンネルを始めることにしました。人気YouTuberの許諾を取って、長尺動画から面白いシーンを切り出してショート動画や10分前後のハイライトを作る、というやつです。

問題はすぐに出てきました。

字幕が必要でした。

---

## GPU地獄に落ちた

当時の私のPCスペックは、10年選手のやつでした。

- CPU: Intel Core i7-6700（2015年世代）
- GPU: GTX 1060 3GB
- RAM: 16GB

WhisperXという字幕生成ツールを動かそうとしたら、起動した瞬間にエラーが出ました。

```
CUDA out of memory. Required 5.2GB, available 2.8GB
```

GTX 1060 3GBのVRAMが3GBしかない（しかも実際に使えるのは2.8GB程度）なのに、WhisperXは5.2GB必要。スペック不足で動かない。

仕方なくCPUモードで動かすと、1つのチャンクを処理するのに36分かかりました。1時間の動画を10分チャンクに分割して6チャンク。つまり字幕生成だけで3時間36分。

さらに話者分離（誰が話しているかを識別する処理）をやろうとしたら、Demucsというツールが必要になりました。これが輪をかけて重い。1時間の動画をDemucsに丸投げしたら、処理の途中でプロセスが死にました。3回試して3回とも死にました。

こうなると選択肢は一つしかありません。

**GPU換装しました。**

RTX 4060 Ti 8GBを約5万円で購入。換装後にWhisperXを動かしたら、36分かかっていた処理が1分で終わりました。36倍速。

「これでようやく本格的に作れる」と思っていました。

その翌週まで。

---

## 「Gemini APIで全部できるじゃないか」

WhisperXで字幕を生成する運用が軌道に乗り始めた頃、試しにGemini APIで音声を丸ごと投げてみました。

結果を見て、固まりました。

**485エントリの字幕が生成されていました。2〜6秒粒度。5人の話者を正確に識別。日本語名で。**

WhisperXで同じ動画を処理すると151エントリでした。10〜30秒の大きなかたまり。話者識別の精度もいまひとつ。

Geminiの出力を見ると、誰が何を話したかが明確にラベルされていて、タイムスタンプも細かい。WhisperXより見やすい字幕が、API1回の呼び出しで出てくる。

費用を計算しました。

**1本あたり約$0.034（約5円）。**

私が5万円で買ったGPUの回収には、同じコストで比較すると147年かかる計算です。

---

## コスト比較

各手法を1本あたりのコストで比較します。

**WhisperX（GPU）**
- コスト: ほぼ0円（GPU代は別途）
- 処理時間: 6〜10分
- 話者識別: 別途処理が必要
- 初期投資: GPU約5万円

**Gemini API**
- コスト: 約$0.034（約5円）
- 処理時間: 1〜2分
- 話者識別: 自動識別あり
- 初期投資: 0円

**AssemblyAI**
- コスト: 約$0.65（約100円）
- 処理時間: 3〜5分
- 話者識別: あり
- 初期投資: 0円

**3エンジン統合（Gemini + AssemblyAI + Deepgram）**
- コスト: 約$0.80（約120円）
- 処理時間: 5〜10分
- 話者識別: あり
- 初期投資: 0円

月10本なら、Geminiだけで月$0.34。3エンジン統合でも月$8程度です。

最小限のPythonコードはこんな感じです：

```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")
audio_file = genai.upload_file("audio.wav")
response = model.generate_content(["字幕をSRT形式で生成してください", audio_file])
print(response.text)
```

5行で動きます。GPUも、Demucsも、チャンク分割も不要です。

## ただし弱点は正直に言わないといけない

「じゃあGemini一択じゃないか」というと、そうでもありません。

**秒精度の限界**: Geminiのタイムスタンプは秒単位です。ミリ秒精度が必要な場合（動画編集で正確なカット位置を指定したいなど）には向きません。どのGeminiモデルを試してもミリ秒は出てきませんでした。

**多声色キャラクターへの対応**: 同じ人物がナレーター・キャラクター・リアクションと複数の声色を使い分けるケースでは、識別精度が落ちます。同一人物が別人扱いされることがありました。

**ハルシネーションのリスク**: まれに、実際には存在しない台詞が出力されることがあります。特に音声が不明瞭な箇所や複数人が同時に話している場面で発生しやすい。検証が必要です。

---

## 最強は3エンジン統合だった

それぞれのエンジンには得意不得意があります。

- **AssemblyAI**: 話者分離が得意。word単位の精密なタイムスタンプが出る
- **Deepgram**: カバレッジが高い。沈黙や雑音の区間でも何かしら出力しようとする
- **Gemini**: コスト最安。話者識別の文脈理解が優秀

3エンジンの出力をタイムスタンプで突合して、各区間の最良候補を選ぶスクリプトを作りました。

```python
# AssemblyAI（ミリ秒単位）をベースに
# Deepgramでカバレッジを補完
# Geminiの話者IDを転写する
```

実際にテストすると、単一エンジンでのカバレッジ95%が、3エンジン統合で98%超になりました。
