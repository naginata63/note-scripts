1万再生なのに登録者23人だった話 — AIに原因を聞いたら身も蓋もなかった


ショートが1万再生を超えた日

チャンネルを開設して2週間。34秒のショート動画が、1万再生を突破しました。

チャンネル初の1万再生です。いいねが131件。コメントも付いている。

「来た。これがバイラルか」

興奮しながら登録者数を確認しました。

**23人。**

1万人が見てくれて、23人。

いや、開設2週間だし、ショートだし、こんなもんか？ いや、でも1万回見られてるのに……？

「これって普通なの？」と思って、AIに聞いてみることにしました。


AIの診断が身も蓋もなかった

前回の記事で競合14チャンネルのデータを取得した仕組みが残っていたので、今度は自分のチャンネルに向けました。

YouTube Data APIで全31動画の詳細データを取得。再生数、いいね数、コメント数、公開日、動画の長さ。全部揃えてAIに渡しました。

「このチャンネルの健康状態を診断してくれ」と。

AIの回答は、こうでした。

**「転換率0.2%。一般的なショートチャンネルの転換率は1〜3%です。10分の1以下。」**

転換率。1万再生を23登録で割った数字です。見た人のうち何%が登録したか。

0.2%。

1,000人が見て、2人しか登録しない。

続けてAIが言ったこと：

**「ショート動画はフィードで消費されます。視聴者はチャンネルページに遷移しません。動画を見て、スワイプして、次の動画を見る。チャンネルの存在を認知すらしていない可能性があります。」**

**「さらに、このチャンネルには登録導線が存在しません。動画内にCTA（チャンネル登録の呼びかけ）がなく、コメント欄にも誘導がありません。」**

まとめると：

- 人間の感想：「1万再生すごい！」
- AIの感想：「再現性なし。バイラル依存体質。登録導線ゼロ。」

身も蓋もない。でも全部事実でした。


31本の内訳を見たら、もっと厳しかった

AIに突きつけられた「バイラル依存」を確認するために、全31動画の再生数を並べてみました。

1万超えは1本だけ。

残り30本の平均再生数は200以下。

つまり、この1本が消えたら1本あたり200再生のチャンネル。それが現実です。

「1万再生突破！」と喜んでいた自分が恥ずかしくなりました。1本バズっただけで、チャンネルとしてはまだ何も始まっていない。

人間はどうしても「良い数字」に目が行きます。1万再生は嬉しい。でもそれは、1万人の前を34秒間通過しただけです。

通過した1万人のうち、チャンネルページに来た人が何人いるのか。来た人のうち、登録した人が23人。**来なかった人は数えることすらできません。**


数字が見えたら、打ち手が見えた

ここからが面白いところです。

問題が「転換率が低い」「登録導線がない」と具体的になった瞬間、打ち手も具体的になりました。

AIが提案したのは**全動画のコメント欄にCTA（登録誘導コメント）を投稿すること**。

動画の中にCTAを入れると「チャンネル登録してね」テロップを毎回付ける必要があります。ショート動画は画面が狭いので邪魔になる。

でもコメント欄なら、動画本体を汚さずに「次の動画も見逃したくない人はチャンネル登録を」と書ける。しかもそのコメントを固定すれば、最初に目に入る。

やることは決まった。じゃあ31本に手動でコメントするのか？

**YouTube Data APIでコメント投稿を自動化しました。**

```python
# 全動画にCTAコメントを一括投稿
for video_id in all_video_ids:
    youtube.commentThreads().insert(
        part="snippet",
        body={
            "snippet": {
                "videoId": video_id,
                "topLevelComment": {
                    "snippet": {
                        "textOriginal": cta_text
                    }
                }
            }
        }
    ).execute()
```

実行したら数分で30本にコメント投稿完了（1本は既にコメント済み）。

ただし、ここで1つ壁にぶつかりました。

**コメントの「固定」はAPIでできません。**

YouTube Data APIには、コメントを投稿する機能はあっても、そのコメントをピン留めする機能がない。Googleのドキュメントを何度読んでも、該当するエンドポイントが存在しない。

なので、コメント投稿はAIが自動でやって、固定は人間がYouTube Studioで手動でやる、というハイブリッド運用になりました。

完全自動にはならなかったけど、31本に手でコメントを打つよりは100倍マシです。


OAuth再認証の罠

ちなみに、コメント投稿のAPI呼び出しでいきなりエラーが出ました。

```
HttpError 403: "Insufficient Permission"
```

原因は、OAuthのスコープ不足でした。

動画のアップロードに使っていたOAuth認証はyoutubeスコープだけで通っていたのですが、コメント投稿にはyoutube.force-sslスコープが追加で必要。

ブラウザで再認証して、スコープを追加して、token.jsonを更新して、再実行。今度は通りました。

こういう「APIの権限まわり」はAIに聞いても正確に教えてくれないことがあるので、結局Googleの公式ドキュメントを読んで解決しました。AIは万能ではない。


収益化まであと何ヶ月か

登録導線を整えたところで、次に気になるのは「収益化までどれくらいかかるのか」です。

YouTubeの収益化条件は2つあって：

- **通常ルート**：登録者1,000人 ＋ 直近12ヶ月の総再生4,000時間
- **ショートルート**：登録者1,000人 ＋ 直近90日のショート視聴回数1,000万回

どちらにしても登録者1,000人は必要です。現在23人。あと977人。

前回の記事で分析した競合データを使って、試算しました。

**楽観シナリオ（月2回バイラル）**

悪だくみ級の1万再生ショートが月に2回出る想定。1回あたり20〜50人の新規登録者が付くと仮定。月40〜100人ペースで、**3〜4ヶ月**。

**現実シナリオ（安定投稿）**

ショート＋ハイライトを週3本ペースで投稿。バイラルに頼らず、検索流入とおすすめ表示で月20〜30人ずつ伸ばす。**6〜8ヶ月**。

**悲観シナリオ（現状維持）**

今と同じペースで週1〜2本、たまにバイラル。月5〜10人。**12ヶ月以上**。

どのシナリオでも「ショート動画のバイラルだけに頼っていたら到達しない」のは共通でした。

AIが最初に指摘した「バイラル依存体質」の意味が、ここで数字として見えます。


「バイラル」と「成長」は別物だった

この分析を通して一番大きかった気づきは、**バイラルと成長は全く別のもの**だということです。

バイラルは「1万人の目の前を通過した」。成長は「通過した人がチャンネルに来て、登録して、次の動画も見にくる」。

ショート動画は拡散力が強い。フィードに流れてきて、面白ければ見る。でもそのまま次の動画にスワイプしてしまう。チャンネルページに遷移する人はごくわずか。

ショートはバイラルの**入口**であって、**ゴール**ではない。

入口から入ってきた人を受け止める場所が必要です。具体的には、8分超のハイライト動画。ショートで「このチャンネル面白いな」と思った人が、ハイライトを見て「ちゃんとした動画も作ってるんだ」と思って登録する。

ショート→興味→ハイライト→登録→継続視聴。この導線を作らないと、いくらショートがバズっても登録者は増えない。

CTA施策はその導線の最初の一歩です。「このチャンネルではこういうことをやっています、登録するともっと見られます」と伝える。


データに先輩も後輩もない

前回の記事（競合14チャンネル分析）と今回の記事で、やったことをまとめます。

今回：

- CTA施策を自動実行
- 収益化タイムラインの試算

全部合わせても、2日で完了しました。費用はほぼゼロ（YouTube Data APIの無料枠内）。

開設2週間、登録者23人の弱小チャンネルでも、5年間・5.7万人の大手チャンネルと**同じデータ、同じ分析**ができます。

数字は誰にでも平等です。問題は、その数字を出すかどうか。

「なんとなく伸びない気がする」で止まるか、「転換率0.2%だから登録導線を作る」と動くか。

AIが出してくれるのは数字と解釈です。行動するのは人間の仕事。でも「何をすべきか」が明確になるだけで、行動のハードルは劇的に下がります。
