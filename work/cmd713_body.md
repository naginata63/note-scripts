4人いるのに2人しか出ない — 自動化の落とし穴はAIじゃなくてif文だった

## 「なんで2人しかいないんだ」

ゲーム実況動画の切り抜きサムネイルを自動生成するパイプラインを作った。
動画には4人出演している。それなのに、自動生成されたサムネには2人しかいなかった。
「なんで？」と思ってコードを読んだら、こんな行があった。

```
if counter >= 2:    selected_chars.append(char)
```

「動画内で2回以上しゃべったキャラだけ採用する」というルールだ。
4人のうち2人が、その動画でたまたま1回しかしゃべっていなかった。だからif文で弾かれた。
でもその「1回だけしゃべったシーン」が、動画最大の盛り上がりだったりする。

## 8ステップのうち、問題が起きたのは全部ルールベースだった

このパイプラインは8ステップで動いている。
問題が起きたのは、ステップ4・5・7。全部ルールベースの箇所だ。
AIを使っているステップ（2・3・5・6）は、むしろ問題なく動いていた。

## 事例①：キャラ選定（STEP4）— 出演回数で足切り

最初の問題が、冒頭に書いたキャラ選定だ。
ロジックはシンプルだった。「動画内でそのキャラが main_speaker として登場した回数をカウントして、2回以上なら採用」。
合理的に聞こえる。よくしゃべったキャラを優先する、というのは直感的だ。
でも「何回しゃべったか」と「動画の面白さへの貢献度」は全然別物だった。
実際にあったのは、こういうケースだ。8シーンある動画で、キャラAが1回だけ登場する。でもその1回が「裏切りシーン」で、動画全体のクライマックスになっている。出演回数は1回。でも「なぜこの動画を見るべきか」を一番よく表しているのはそのシーンだ。
if文はそういう文脈を読めない。回数しか見ていないから。

**修正したこと：** 自作ツールに全シーンの内容を渡して「この動画の核となるキャラは誰か」を判断させた。修正後は4人全員が登場する案が自然に生成された。

## 事例②：レイアウト選定（STEP7）— 人数→固定マップ

次はレイアウトだ。こんなルールがあった。

```
if char_count == 2:    layout = "layout_A"
elif char_count == 3:    layout = "layout_D"
elif char_count == 4:    layout = "layout_E"
```

「登場キャラが2人ならA配置、3人ならD配置、4人ならE配置」。
これも最初は問題ないように見えた。人数が決まれば配置が決まる。シンプルでいい。
でも「裏切りシーン」も「協力シーン」も、4人いれば同じlayout_Eになる。
裏切りなら対決配置（主役VSその他）の方が映える。協力なら横並び集合配置の方がいい。シーンの感情に合ったレイアウトの使い分けが、人数ルールだと一切できない。

**修正したこと：** レイアウト選定も自作ツールに任せた。シーンの内容（字幕テキスト）を渡して「このシーンに最適な配置は何か」を判断させるようにした。

## 事例③：表情マッチング（STEP5）— AIの選択をルールが上書きした

3つ目が一番厄介だった。
表情マッチングのStepは、Embeddingを使っている。字幕テキストをベクトル化して、9種類の表情ラベル（笑顔・驚き・怒り・したり顔など）とのコサイン類似度で最適な表情を選ぶ仕組みだ。
このステップ自体は正しく動いていた。「このセリフには驚き顔が合う」という判断を、ちゃんとやっていた。
問題は、その後に呼ばれる with_variant() 関数だ。

```
def with_variant(expression, rank):
    if expression in ["smile", "surprise", "angry"]:
        return f"{expression}_r3"  # 最高ランクを強制
    return f"{expression}_r{rank}"
```

「笑顔・驚き・怒りは必ずr3（最高強度）を使え」というルールを強制していた。
r3は「最も強い表情」だ。ただし、全キャラにr3が用意されているわけではない。あるキャラはr3がない。そのキャラで驚き顔を指定すると、r3がないから別の表情にフォールバックして、意図しない顔になる。
**AIが正しく選んだ結果を、ルールが上書きして壊した。**
しかも厄介なのは、QCで見つけにくいことだ。「笑顔のフォールバックが0件」という基準でチェックしていたら、「驚きのフォールバック」は素通りする。症状の名前が違うだけで、根本原因は同じなのに。

**修正したこと：** 使用前に実在するファイルを確認して、r3→r2→r1の順でフォールバックするよう変更した。存在しない表情を指定しない、という当然のことをするだけで直った。

## おまけ：サイズ問題 — 人間が5秒で気づいたこと

これはルールでもAIでもなく、パラメータの話だ。
生成されたサムネイルを競合チャンネルのサムネイルと並べて見た。一言で言われた。
「小さい」
全レイアウトのキャラの高さを確認したら、最大でも画面の68%だった。競合は80〜90%。画面いっぱいにキャラをかぶせているところが多い。
AIにスコアリングさせていた。「競合との類似度」を測るスコアだ。でもそのスコアは「競合と並べた時の印象」を測っていない。
「なんか小さい」という感覚は、数値にならない。人間が競合を見て「もっとデカく」と一言言えば一発で解決するが、AIがそれを自律的に気づくのは難しい。

## 結論：ルールベースが安全な場所、AIに任せるべき場所

この経験から整理した分類がこれだ。
ルールベースが安全なのは、「全パターンが事前に列挙できる場合」だけだ。
「何回しゃべったか」でキャラを選ぶ、「人数」でレイアウトを決める、「笑顔なら必ずr3」と固定する。これらは全部「パターンを事前に定義しようとした」試みだ。
でも「どのキャラが面白いか」「どの配置がこのシーンに合うか」は、列挙できない。動画の中身を読まないと答えが出ない問いだ。そこにif文を書いた時点で、答えは出ない。

## AIとif文の使い分けを間違えると、AIじゃない箇所が全部壊れる

今回学んだことをひとつに絞るなら、これだ。
「AI部分が問題ないのに、ルールベース部分が全部問題だった」という事実は、最初に気づいたとき少し驚いた。AIを導入したパイプラインで、問題はAIじゃないところで起きていた。
理由を考えると当然で、AIは文脈を読む。if文は文脈を読まない。
「このシーンに4人出ているが、全員同じくらい重要か？」をAIに聞けば、ちゃんと答える。でもif文に聞くと「2人以上か以下かだけ教えろ」という返事しか返ってこない。
そのギャップが、サムネに2人しか出ない問題として現れた。
ルールベースを使うとき、「これは本当に全パターン列挙できるか？」を一度立ち止まって確認するようになった。できるなら使う。できないならAIに任せる。
それだけで、パイプラインの品質が変わった。
